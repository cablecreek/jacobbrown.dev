By learning nvim I have realise that having your docs within your editor is a preferable way to go. There is no context switching, you staying in the same program, and concepts like `<leader>K` and `</leader>gr` feel far more natural.

At present, the Terraform provider documentation is broswer based...
Let's fix that --> [`tf-docs.nvim`](https://github.com/cablecreek/tf-docs.nvim)
